---

COMPONENT A — PII DETECTOR

Scan any string and detect:
  Student names     DB cache, refresh every 5 minutes
  Teacher names     same DB cache
  Email addresses   regex
  Phone numbers     regex
  Roll numbers and admission IDs   regex + context

Return every match with type, value, and exact
character start and end positions.

---

COMPONENT B — TOKENIZER

Replace each PII hit with a readable token:
  STUDENT_A91F   TEACHER_C22B   EMAIL_F301   PHONE_9A12

HMAC-SHA256 generation using a secret from env vars.
Same entity always gets the same token per session.

piiMap lives in request memory only.
Never written to DB, file, cache, or logs.

Reverse function restores tokens in LLM responses.

---

COMPONENT C — OCR SANITIZER

Strip identity from answer sheet OCR text:
  "My name is ..."   "Roll No ..."
  "Class 10A ..."    "I am ..."
  Any name in student cache

Replace with [STUDENT_IDENTIFIER].
Run full PII detector as a second pass after stripping.

---

COMPONENT D — PRIVACY GUARD

Central orchestrator. Every AI call passes through here.

Conversational query flow:
  detect -> mask -> send -> role check -> unmask
  -> return -> discard piiMap

OCR evaluation flow:
  sanitize -> mask -> send -> role check -> unmask
  -> return -> discard piiMap

Unmasking rules:
  Student   own data only
  Teacher   students in their assigned classes only
  Admin     everything

Throw ACCESS_DENIED if a role exceeds its permission.

---

COMPONENT E — CENTRAL LLM REGISTRY

Single file. Single source of truth.
No model name, temperature, token limit, or system
prompt may exist anywhere else in the codebase.

Call types to define:
  QUERY       conversational analytics
  EVALUATE    answer sheet OCR evaluation
  INSIGHT     class-wide insight generation
  HOMEWORK    homework feedback
  RANKING     student ranking analysis

For each: model, temperature, max input tokens,
max output tokens, timeout, full system prompt.

ASK ME what model and limits to use.

---

COMPONENT F — AI GATEWAY

Only file allowed to import OpenAI.
All config comes from registry.
Privacy guard runs before every call.
Final regex sweep before sending — throw UNSAFE_PAYLOAD
if anything suspicious found.

Safe audit log only: request_id, timestamp, call_type,
model, token_usage, latency_ms.
Never log prompt, response, OCR, or piiMap.

Public functions only:
  askQuestion(prompt, userContext)
  evaluateAnswerSheet(ocrText, rubric, userContext)
  generateClassInsight(analyticsData, userContext)
  analyseHomework(homeworkData, userContext)
  generateRankingInsight(rankingData, userContext)

---
